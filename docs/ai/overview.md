## Обзор AI-интеграции в scrcpy

Этот репозиторий использует AI‑агента Cursor для:

- ускорения разработки C‑клиента (`app/`) и Android‑сервера (`server/`);
- поддержки правок документации в `doc/` и `docs/ai/`;
- безопасного рефакторинга и улучшения качества кода.

AI‑инструкции формализованы в файле `.cursorrules` в корне проекта и в документах в `docs/ai/`.

## Задачи, для которых рекомендуется AI

- Небольшие изменения в существующих модулях без изменения внешнего API;
- исправление багов (crash, утечки ресурсов, некорректная обработка ошибок);
- локальный рефакторинг (упрощение функций, выделение вспомогательных методов);
- улучшение логирования и сообщений об ошибках;
- дополнение/уточнение документации в `doc/*.md`;
- подготовка патчей под чётко описанную задачу.

Для любых изменений публичного поведения (CLI‑опции, протокол клиент–сервер, скрипты релиза) AI обязан запросить подтверждение у пользователя (см. раздел `ask_confirmation` в `.cursorrules`).

## Общий рабочий процесс с AI‑агентом

1. **Постановка задачи.** Описать цель, контекст и ограничения. По возможности указать:
   - файлы/папки, которых касается задача;
   - текущее и желаемое поведение;
   - как проверить результат (команда сборки/запуска).
2. **Ответ AI.** Агент обязан выдать:
   - план действий;
   - перечисление затронутых файлов;
   - диффы или целевые фрагменты кода;
   - пояснение ключевых решений;
   - шаги проверки.
3. **Проверка разработчиком.**
   - просмотреть предложенные изменения;
   - запустить сборку/тесты по шагам проверки;
   - при необходимости скорректировать задачу и повторить цикл.

## Команды для сборки и запуска (кратко)

> Все команды должны указываться в блоках `code`:

### Клиент (native C, desktop)

- Сборка с Meson/Ninja:

```bash
meson setup builddir
ninja -C builddir
```

- Пример запуска (если бинарник называется `scrcpy` и лежит в `builddir/app/`):

```bash
./builddir/app/scrcpy
```

### Android‑сервер

- Сборка сервера с помощью Gradle wrapper:

```bash
cd server
./gradlew assembleDebug
```

### Тесты

- Тесты сервера (Java):

```bash
cd server
./gradlew test
```

- Клиентские тесты/прогоны (см. скрипты в `release/`):

```bash
./release/test_client.sh
./release/test_server.sh
```

Подробные инструкции по сборке и использованию см. в `README.md` и `doc/build.md`.
